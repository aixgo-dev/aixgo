# Introducing Aixgo Sessions: AI Agents That Remember

**Release Date**: February 8, 2026
**Version**: 0.3.0-alpha

We're excited to announce Aixgo Sessions - a built-in session persistence system that gives your AI agents memory, context, and continuity out of the box.

## The Problem with Stateless Agents

Most AI agent frameworks treat memory as an afterthought. You build your agent, deploy it, and then realize:

- Users have to re-explain context every conversation
- There's no way to resume interrupted workflows
- Multi-turn interactions lose critical information
- You're stuck building custom persistence from scratch

This is because most frameworks come from a serverless, Lambda-style background where state is temporary by design.

## Why Sessions by Default?

Aixgo takes a different approach. **Sessions are the default**, not an opt-in feature.

When you create an Aixgo agent, conversation history persists automatically. Your agents remember previous exchanges, can reference earlier context, and survive application restarts. No configuration required.

```go
// Sessions are automatic - just provide a user ID
resp, _ := rt.Call(ctx, "assistant", msg, aixgo.WithUser("user-123"))

// Context persists across calls - the agent remembers
```

For the rare cases where you need stateless processing (batch ETL, classification pipelines), you can explicitly opt out:

```yaml
agents:
  - name: batch-classifier
    role: classifier
    session_mode: disabled  # Explicit opt-out
```

## What's Included in v0.3.0

### Core Session Management

- **Create, Resume, List, Delete** - Full session lifecycle management
- **Automatic persistence** - JSONL file-based storage that survives restarts
- **User isolation** - Sessions are scoped to users and agents

### Checkpoints

Save and restore conversation state at any point:

```go
// Create a checkpoint before a risky operation
checkpoint, _ := sess.Checkpoint(ctx)

// ... something goes wrong ...

// Restore to the saved state
sess.Restore(ctx, checkpoint.ID)
```

### Session-Aware Agents

ReAct agents now implement `SessionAwareAgent`, gaining access to full conversation history during execution:

```go
type SessionAwareAgent interface {
    ExecuteWithSession(ctx context.Context, input *Message, sess Session) (*Message, error)
}
```

### Runtime Integration

The runtime's new `CallWithSession` method handles everything automatically:

```go
// Automatically appends input/output to session history
result, err := rt.CallWithSession(ctx, "assistant", msg, sessionID)
```

## Quick Start

### Installation

```bash
go get github.com/aixgo-dev/aixgo@v0.3.0-alpha
```

### Basic Usage

```go
package main

import (
    "context"
    "github.com/aixgo-dev/aixgo"
    "github.com/aixgo-dev/aixgo/pkg/session"
)

func main() {
    ctx := context.Background()

    // Create session infrastructure
    backend, _ := session.NewFileBackend("~/.aixgo/sessions")
    mgr := session.NewManager(backend)
    defer mgr.Close()

    // Create a session
    sess, _ := mgr.Create(ctx, "assistant", session.CreateOptions{
        UserID: "user-123",
    })

    // Append messages
    msg := agent.NewMessage("user", map[string]string{
        "content": "Hello! My name is Alice.",
    })
    sess.AppendMessage(ctx, msg)

    // Later: resume the session
    resumed, _ := mgr.Get(ctx, sess.ID())
    messages, _ := resumed.GetMessages(ctx)
    // Alice's introduction is still there!
}
```

## Performance

Sessions add minimal overhead:

| Operation | Latency |
|-----------|---------|
| Create session | <1ms |
| Append message | <1ms |
| Get 100 messages | <5ms |
| Checkpoint | <1ms |

The JSONL storage format is append-only, ensuring durability without sacrificing performance.
